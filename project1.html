Nexivus Point of Sale System

A robust Spring Boot-based Point of Sale system with secure JWT authentication, RESTful APIs, and end-to-end sales and inventory management features.

Project Overview

Nexivus is a full-featured inventory management system built with Spring Boot, featuring modern authentication, comprehensive API design, and scalable architecture.

Secure Authentication

Implements JWT-based authentication with Spring Security, including token generation, validation, and automatic admin user creation for initial setup.

Complete CRUD Operations

Full Create, Read, Update, Delete functionality for all entities including users, categories, items, orders, and stock management with proper validation.

RESTful API Design

Well-structured REST endpoints following industry standards with proper HTTP status codes, request/response handling, and comprehensive error management.

API Documentation

Comprehensive RESTful API endpoints

Authentication

POST /auth/login

Authenticate user and receive JWT token

User Management

GET /users

Retrieve all users

POST /users

Create new user with username, fullname, and password

Category Management

GET /category

Retrieve all item categories

POST /category

Create new category with name

Item Management

GET /items

Get all inventory items (requires JWT token)

POST /items

Add new inventory item with name, description, price, and categoryId

PUT /items/{id}

Update existing item information

DELETE /items/{id}

Delete item from inventory

Stock Management

GET /stock

View current stock levels for all items

POST /stock

Create initial stock entry for item

PUT /stock/addto

Add quantity to multiple items (requires JWT token)

PUT /stock/getfrom

Remove quantity from multiple items (requires JWT token)

PUT /stock

Update stock quantity for specific item (requires JWT token)

Order Management

GET /orders

Retrieve all orders (requires JWT token)

POST /orders

Create new order with array of item IDs (requires JWT token)

Authentication Notes

JWT Token Required

Most endpoints require a valid JWT token in the Authorization header as Bearer token.

Authorization: Bearer <your-jwt-token>
